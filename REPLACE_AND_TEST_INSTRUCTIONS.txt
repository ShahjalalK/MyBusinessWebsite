TrackFlow Pro v18.95.4 — Self-hosted Email Open/Click Tracking Fix

Replace these files:

app/api/trackflow/[...action]/route.ts
PROJECT_CONTEXT_README.md

Why:
- The existing email send flow relied mainly on Brevo webhook open/click events.
- If Brevo webhook is not configured, delayed, or does not send tags as expected, dashboard open/click counters do not update.
- This patch adds TrackFlow-owned open and click tracking URLs into every sent email.

After replace:

npm run build
npm run dev

Test:
1. Send a test email from Send Email tab to a different mailbox.
2. Inspect the report/link URL in the received email.
3. It should start with:
   https://trackflowpro.com/api/trackflow/track/click?...
4. Click the link.
5. Open Leads tab and refresh.
6. The lead should show clicked / click_count increased.
7. Open tracking may depend on the recipient mail client loading images.

Note:
- Brevo webhook tracking remains supported.
- This is an additional fallback, not a replacement.
